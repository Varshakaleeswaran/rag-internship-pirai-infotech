📌 Project Overview

This project implements Retrieval-Augmented Generation (RAG) using ChromaDB as the vector database.

RAG combines:

Large Language Model (LLM) – to generate answers

Retriever (ChromaDB) – to fetch relevant documents

Embedding model – to convert text into vectors

🚀 The goal is to build a system where:

A user asks a question

Relevant documents are retrieved from ChromaDB

The LLM generates an answer using those retrieved documents

This improves accuracy, reduces hallucinations, and makes responses knowledge-grounded.

🎯 Objectives

Understand RAG architecture from scratch

Learn embeddings and vector similarity search

Set up and use ChromaDB for document storage

Build a simple question–answer RAG application

Document daily internship progress

Publish complete work on GitHub

🧩 What is ChromaDB?

ChromaDB is an open-source vector database designed for:

Storing embeddings

Performing similarity search

Fast retrieval in RAG systems

It is simple, lightweight and easy for beginners.
